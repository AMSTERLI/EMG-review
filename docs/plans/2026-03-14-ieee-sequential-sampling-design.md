# IEEE 年度 CSV 顺序采样设计

**目标：** 对 `original-csv/2000.csv` 到 `original-csv/2026.csv` 这 27 个 IEEE CSV 文件，按各文件中的原始文献顺序进行 30% 垂直取样，小数向上取整，最后按年份顺序合并为一个总文件，并输出每年的取样数量。

## 背景

当前仓库已有：

- `random_sample.py`：对合并后的数据做随机抽样
- `merge_csv.py`：合并 PubMed 与 IEEE 数据

本需求不属于“随机抽样”或“数据库合并”，而是对已按年份拆分的 IEEE CSV 做顺序采样，因此应使用独立脚本，避免混淆现有流程。

## 设计选择

### 方案 1：改造 `random_sample.py`

优点是复用现有脚本；缺点是现有脚本语义为随机分层抽样，与“按文件原始顺序取前 30%”逻辑不同，职责会变得不清晰。

### 方案 2：新建独立脚本

新建一个专用脚本，例如 `sequential_sample_ieee_years.py`，只处理 `original-csv/2000.csv` 到 `original-csv/2026.csv`。这是推荐方案，因为它与现有流程解耦、便于重复运行、也更容易验证。

### 方案 3：将逻辑加入 `merge_csv.py`

不推荐。`merge_csv.py` 属于数据清洗与跨库合并阶段，将顺序采样塞进去会导致脚本职责过重。

## 推荐方案

采用方案 2：新增独立脚本和独立测试。

## 数据流

1. 按年份从 `2000` 遍历到 `2026`
2. 读取每个 `original-csv/<year>.csv`
3. 计算该年应取样本数：`ceil(len(df) * 0.3)`
4. 使用 `df.iloc[:sample_count]` 保留文件中的原始顺序
5. 将每年的样本加入列表
6. 将所有年份样本按年份顺序拼接
7. 输出合并文件到 `original-csv/ieee_emg_sampled_30_percent_2000_2026.csv`
8. 在终端打印每年原始数量与取样数量，并打印总体统计

## 错误处理

- 若某一年的 CSV 缺失：打印警告并跳过
- 若 CSV 为空：该年样本数为 `0`
- 若编码读取失败：尝试常见回退编码，保证尽可能稳定读取历史导出文件
- 若某年列结构与其他年份存在差异：使用 `pandas.concat(..., sort=False)` 保留并集列

## 测试策略

采用最小可验证测试：

- 测试样本数按 30% 且向上取整
- 测试采样保持原始行顺序，而不是随机抽样
- 测试多年份输入时合并结果按年份顺序输出

## 成功标准

- 能从 `2000.csv` 到 `2026.csv` 自动完成处理
- 合并文件成功生成
- 每年的取样数量可在终端直接查看
- 采样规则与用户定义完全一致：按 CSV 中原顺序、30%、向上取整
