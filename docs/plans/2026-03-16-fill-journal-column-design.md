# 为 Sampled_Papers_Stratified 填充期刊列设计

**目标：** 为 `PDF_Dataset/Sampled_Papers_Stratified.csv` 新增 `Journal` 列，并按论文标题联网查询期刊名称后写回表格。

## 背景

当前 CSV 只有：

- `ID`
- `Title`
- `Abstract`
- `Year`
- `Database`
- `Status`

文件中没有 DOI、PMID、ISSN 或期刊字段，因此无法仅靠本地数据推断期刊名，必须通过标题联网查询。

## 方案比较

### 方案 1：只用一个公开来源

可以直接调用 `Crossref` 按标题搜索，优点是实现简单；缺点是部分医学论文命中率不稳定。

### 方案 2：按数据库分层回退查询

先用 `Crossref` 按标题查期刊名；若结果不可靠或为空，再对 `Database=PubMed` 的记录尝试 `PubMed` 标题搜索。这个方案兼顾通用性与命中率，推荐使用。

### 方案 3：手工下载元数据后再合并

准确率可控，但流程重、重复劳动多，不适合当前批量补充字段需求。

## 推荐方案

采用方案 2：

1. 读取 CSV
2. 若不存在 `Journal` 列则新增
3. 对已有 `Journal` 值的行直接跳过，支持断点续跑
4. 先按标题调用 `Crossref`
5. 若无结果且 `Database=PubMed`，再调用 `PubMed`
6. 将查询到的期刊名写入 `Journal`
7. 查不到则留空

## 数据规则

- 匹配键：`Title`
- 新列名：`Journal`
- 已有非空 `Journal` 值时不重复查询
- 标题为空时直接跳过
- 查不到期刊时保留空值

## 错误处理

- 网络错误：打印 warning，当前行留空，继续后续行
- API 返回空结果：留空
- 速率限制：脚本提供可配置等待时间，避免请求过快
- 中途终止：已写入列的结果可用于下次续跑

## 输出设计

- 默认原地更新 `PDF_Dataset/Sampled_Papers_Stratified.csv`
- 在终端打印：
  - 总行数
  - 已跳过数
  - 成功填充数
  - 失败或未命中数

## 成功标准

- CSV 成功新增 `Journal` 列
- 可重复运行且不会重复查询已填充行
- 对大部分记录能自动写入期刊名
- 查不到的记录保持空值，不中断全流程
